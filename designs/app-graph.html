Fluxos de Dados
Como os dados transitam entre UI, banco e triggers nos principais fluxos de negócio.
Fluxo — Agendamento completo (status machine)
sequenceDiagram actor Prof as Profissional participant UI as Agenda UI participant DB as Supabase DB participant TG as Trigger SQL Prof->>UI: Abre modal de novo agendamento UI->>DB: Promise.all(clientes, servicos, empresa_membros) DB-->>UI: Dados para SearchSelect Prof->>UI: Preenche e confirma UI->>DB: INSERT agendamentos (status=agendado) UI->>DB: INSERT agendamento_servicos[] DB-->>UI: id do agendamento Note over UI,DB: Dia do atendimento Prof->>UI: Drag ou clique → status=concluido UI->>DB: UPDATE agendamentos SET status='concluido' (optimistic UI) DB->>TG: gerar_comissao() dispara TG->>DB: SELECT empresa_membros WHERE user_id = profissional_id TG->>DB: INSERT comissoes (valor × percentual_comissao / 100) Prof->>UI: Clica em "Avaliar" UI->>DB: INSERT avaliacoes (nota, comentario)
Fluxo — Comanda / Venda Rápida
sequenceDiagram actor At as Atendente participant C as Comanda UI participant DB as Supabase DB At->>C: Seleciona cliente (SearchSelect) C->>DB: SELECT agendamentos concluídos sem comanda + servicos + produtos DB-->>C: Itens disponíveis At->>C: Adiciona serviços e/ou produtos At->>C: Define desconto (%) At->>C: Escolhe pagamento (dinheiro/pix/cartão + bandeira + parcelas) C->>DB: INSERT comandas {clientes_id, total} C->>DB: INSERT comanda_itens[] {tipo, referencia_id, preco, qtd} C->>DB: INSERT pagamentos {valor, valor_liquido, metodo, bandeira, parcelas} C->>DB: INSERT vendas + venda_itens[] (se houver produtos) C->>DB: INSERT estoque_movimentos {tipo=saida, produto_id, quantidade} C->>DB: UPDATE agendamentos SET comanda_id (vincula agendamento) DB-->>C: ✅ Toast de confirmação
